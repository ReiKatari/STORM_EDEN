<h4 align="center"><b>STORM EDEN</b> — это оптимизированный эмулятор Nintendo Switch с открытым исходным кодом. Представляет собой модифицированную версию проекта <b>Eden</b> (форк <b>Yuzu</b> и <b>Sudachi</b>) с улучшенной производительностью, полной локализацией и встроенными инструментами фонового управления дополнениями.</h4>

---

## 📊 Сравнение STORM EDEN с оригинальной версией (eden-emu/eden)

Ниже приведен полный список изменений, добавленных в сборку **STORM EDEN** по сравнению с оригинальным репозиторием [git.eden-emu.dev/eden-emu/eden](https://git.eden-emu.dev/eden-emu/eden):

### 1. Фоновый менеджер дополнений (DLC)
* **Автоматический кэшер Tinfoil**: При запуске эмулятор считывает локальный кэш `~/.switch/titledb.json` или в фоне загружает свежую базу дополнений (`https://tinfoil.media/repo/db/titles.json`).
* **Динамическое управление приоритетом (Priority Bump)**: При нажатии кнопки дополнений во время фоновой загрузки приоритет сетевого запроса автоматически повышается до Высокого (High Priority). Кнопка временно блокируется с индикацией `"ЗАГРУЗКА..."` и плавно открывает диалоговое окно DLC сразу по готовности.
* **Интерактивный счетчик DLC в статус-баре**: Кнопка **ДОПОЛНЕНИЯ (активно/всего)** (например, `ДОПОЛНЕНИЯ (5/12)`) динамически отображает количество установленного контента для запущенной игры и обновляется при старте/остановке эмуляции.
* **Контекстное меню копирования**: В список дополнений добавлено контекстное меню для удобного копирования названий DLC.

### 2. Безопасность и стабильность (Исправление вылетов)
* **Предотвращение критического вылета на старте**: Исправлен сбой, при котором вызов `GetApplicationProcessProgramID()` приводил к падению из-за отсутствия запущенного процесса в ядре во время инициализации или старта игры.
* **Устойчивость к битым файлам**: Внедрены проверки на нулевые указатели (`nullptr`) при получении DLC из менеджера контента, что предотвращает вылеты при наличии поврежденных дополнений.

### 3. Полная локализация всего интерфейса
* **Локализация**: Выполнена **полная локализация всего интерфейса** эмулятора на русский язык.
* **Локализация патчей и DLC**: В общем списке игр в колонке «ДОПОЛНЕНИЯ» теперь выводится корректное `"Версия"` вместо `"Update"`, `"Дополнения"` вместо `"DLC"`. Строка `"Unknown DLC [%1]"` локализована как `"Неизвестное дополнение [%1]"`.
* **Графические настройки**: Исправлена ошибка перевода `"Расширения Vulcan"` в расширенных настройках графики, теперь отображается корректное **`"Расширения Vulkan"`**.
* **Параметры рендеринга**: Полностью локализованы параметры **`Fast GPU Time`** (как **`"Быстрое время GPU"`**) и **`anti_flicker`** (как **`"Подавление мерцания"`**) для русского и английского языков.

### 4. Оптимизация производительности и устранение фризов
* **Кэширование метаданных сканируемых папок**: Ранее при каждом запуске эмулятор считывал и декодировал иконки и названия всех игр (включая файлы более 5 ГБ и сжатые NSZ/XCZ) непосредственно из самих файлов, что приводило к долгим зависаниям интерфейса при открытии. Теперь все метаданные сканируемых папок полностью кэшируются на диске, обеспечивая мгновенный запуск интерфейса эмулятора.
* **Оптимизация оперативной памяти (RAM) при распаковке NSZ**: Ранее при запуске игр с обновлениями большого объема (от 8 ГБ) в формате NCZ эмулятор пытался полностью загрузить и распаковать весь файл в оперативную память перед патчингом BKTR, выделяя до 14+ ГБ ОЗУ, что вызывало OOM-вылеты (`bad_alloc`) или полное зависание компьютера. Теперь распаковка производится поблочно (блоками по 4 МБ) во временный файл на диске с последующим мгновенным произвольным доступом, снижая потребление ОЗУ во время запуска до стабильных 4 МБ.
* **Устранение I/O бутылочного горлышка**: Полностью отключена тяжелая фоновая запись логов в файл `debug_log.txt`, которая перегружала накопитель и вызывала микрофризы во время игры.
* **Оптимизация SimpleIni**: Отключен избыточный отладочный вывод библиотеки SimpleIni (вывод 800+ строк конфигурационных ключей при каждом автосохранении настроек), что избавило эмулятор от лагов и задержек при старте игр.
* **Оптимизация процессора (CPU)**: Уровень логирования служебных операций с файловой системой (`nsp.cpp`, `partition_filesystem.cpp`, `submission_package.cpp`, `content_archive.cpp`) понижен с `LOG_CRITICAL` до `LOG_DEBUG` для уменьшения процессорного оверхеда.

### 5. Дополнительные исправления (Bugfixes)
* **Поддержка смещения AOC ID**: Добавлена корректная обработка DLC с базовым смещением ID `0x1000` в функции `CheckAOCTitleIDMatchesBase`.
* **Исправление инициализации настроек**: Тип настроек `Settings::VramUsageMode` объявлен в качестве метатипа Qt (`Q_DECLARE_METATYPE`), что устранило проблемы при первом запуске эмулятора.

### 6. Улучшения Android-версии
* **Автоматическое скачивание и кэширование TitleDB**: При запуске Android-версии в фоновом потоке проверяется и обновляется база данных Tinfoil (`titles.json`), которая кэшируется локально по пути `<userDirectory>/.switch/titledb.json` не чаще раза в сутки.
* **Отображение неактивных DLC**: В окне «Дополнения» эмулятора теперь выводится полный список DLC (включая неактивные/неустановленные, сгенерированные динамически по алгоритму смещения AOC ID) с пометкой `"НЕ АКТИВНО"` и отключенными переключателями.
* **Быстрое копирование названий DLC**: Добавлена возможность копирования названия дополнения в буфер обмена по долгому нажатию на элемент списка.
* **Отображение версий игр**: Текущие версии игр выводятся прямо в общем списке игр эмулятора (рядом с разработчиком, например, `"Nintendo | v1.0.3"`) и в заголовке свойств игры (`"Игровое время: Xч Yм | Версия: 1.0.3"`). Принудительное обновление кэша версии происходит при каждом открытии свойств игры.

### 7. Переход на SDL3 и исправление детекции геймпадов (КРИТИЧНО)
* **Переход на SDL3**: Выполнен полный переход подсистемы ввода, звука и консольного лаунчера на **SDL3** (заменены библиотеки `sdl2` -> `sdl3` / `sdl3-ci`).
* **Исправление детекции геймпадов**: Полностью решена проблема, при которой STORM EDEN на Windows вообще не определял геймпады (включая Nintendo Switch Pro Controller) даже при закрытом Steam. Благодаря нативной обработке устройств в SDL3 геймпады теперь определяются мгновенно и без дополнительных прослоек.

## 🛠️ Зависимости для сборки

Для сборки STORM EDEN требуются:

* **Компилятор**: MSVC 19.40+ (Visual Studio 2022 v17.10+) / GCC 12+ / Clang 15+.
* **Инструменты**: CMake 3.22+ и Ninja.
* **Qt Framework**: Qt 6.9.0 или новее (модули `Core`, `Widgets`, `Network`, `Concurrent`).
* **Библиотеки**: OpenSSL 3.0+, Vulkan SDK 1.3+, Boost 1.79+, SDL3.

## 💻 Сборка проекта

### Windows (Быстрый запуск)
Выполните скрипт автоматической компиляции:
```cmd
build_fast.bat
```

### Вручную через CMake
```bash
cmake -B build -G "Visual Studio 17 2022" -A x64 -DCMAKE_BUILD_TYPE=Release
cmake --build build --config Release
```

Готовый файл сборки будет расположен по пути `build/bin/Release/eden.exe`.
